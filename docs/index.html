Design

Built around
correctness.

Every decision in this library prioritises type safety and predictable behaviour over convenience shims.

01
No Roblox Instance Overhead

Signal uses a plain Luau linked list — no BindableEvents, no Instance.new, no memory cost beyond the connection nodes themselves. Fires synchronously in the same thread unless you call FireDeferred.

02
Safe Mid-Fire Disconnects

Disconnecting a handler while its Signal is actively firing won't corrupt iteration. Pending removals are queued and flushed once the fire cycle ends, so you never skip or double-fire a handler by accident.

03
Typed Cancellation Chains

Promise cancellation propagates through parent chains. The onCancel hook lets you register cleanup (disconnect a connection, cancel a task.delay) that fires immediately if the Promise is already cancelled at hook time.

04
Schema-Validated Networking

Net.defineNamespace enforces payload shape at both call site and receive site. Mismatched fields throw before the packet ever reaches the Remote, not silently after — catching protocol drift the moment it happens.

05
Middleware Pipeline

Net ships with built-in rateLimit and logger middleware. Add your own auth checks or analytics by pushing to the middleware stack — each layer calls next() or aborts the event by not calling it.

06
Task-Aware Scheduling

All async operations use task.spawn and task.defer — never raw coroutine.wrap or spawn(). This keeps execution order consistent with Roblox's deferred signal model and avoids legacy scheduler bugs.

Usage

Quick examples.

All three packages follow the same idiom — require, type, use.

local Signal = require(ReplicatedStorage.Packages.Signal)

-- Create a typed signal
local playerDied = Signal.new() :: Signal.Signal<Player, string>

-- Connect a permanent handler
local conn = playerDied:Connect(function(player: Player, cause: string)
  print(player.Name, "died from", cause)
end)

-- Fire with arguments
playerDied:Fire(somePlayer, "fall damage")

-- Wait for the next fire (yields the current coroutine)
local p, cause = playerDied:Wait()

-- Clean up when done
conn:Disconnect()
playerDied:Destroy()
local Promise = require(ReplicatedStorage.Packages.Promise)

-- Load player data with retry + timeout
local p = Promise.retry(function()
  return Promise.new(function(resolve, reject, onCancel)
    local thread = task.spawn(function()
      local ok, data = pcall(DataStore.GetAsync, DataStore, key)
      if ok then resolve(data) else reject(data) end
    end)
    onCancel(function() task.cancel(thread) end)
  end)
end, 3, 1)  -- 3 attempts, 1s between each

p:andThen(function(data) applyData(data) end)
 :catch(function(err) warn("Load failed:", err) end)
 :finally(function() hideLoadingScreen() end)

-- Race a load against a 5-second timeout
local result = Promise.race({
  fetchInventory(),
  Promise.delay(5):andThen(function() return {} end),
}):expect()
-- shared/Network.luau (required on both sides)
local Net = require(ReplicatedStorage.Packages.Net)

Net.useMiddleware(Net.Middleware.rateLimit(20))  -- 20 events/s per player
Net.useMiddleware(Net.Middleware.logger())

return Net.defineNamespace("Inventory", function(define)
  define.event("DropItem", { itemId = "string", slot = "number" })
  define.request("GetInventory", {}, { items = "table" })
end)

-- server/InventoryService.luau
Inventory.DropItem:On(function(player, data)
  InventoryManager:drop(player, data.itemId, data.slot)
end)
Inventory.GetInventory:Handle(function(player)
  return { items = InventoryManager:getAll(player) }
end)

-- client/InventoryUI.luau
Inventory.DropItem:Fire({ itemId = "sword_01", slot = 3 })
local inv = Inventory.GetInventory:Invoke({})  -- typed response
